## Graceful-Degradation Waiver

Runtime negative testing on an external CLI runner was blocked by host model policy:
`claude-cli/claude-sonnet-4-6` is not allowed for agent `main`. This is recorded as a
deferred live-runtime test, not a silent skip.

Provider-neutral property verified for this release:

- Bounded mode registers no hooks, so it is identical across embedded runners,
  external CLI runners, and non-dispatching harnesses.
- Enhanced hook-dependent paths are opt-in and experimental. If a harness does not
  dispatch in-process lifecycle hooks, the hooks simply do not run.
- Unit coverage simulates enhanced hooks being registered but never fired and proves
  the three core tools (`native_memory_search`, `native_memory_fetch`,
  `native_memory_answer`) still succeed without throwing.
- Documentation now describes hook dispatch generically rather than naming a specific
  provider as the required negative case.

Re-run the live external-runner negative when a model/runner allowlist permits a
controlled `claude-cli`, Codex CLI, or equivalent external-CLI turn.
